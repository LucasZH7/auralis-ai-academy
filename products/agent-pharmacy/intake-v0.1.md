# Agent Pharmacy Intake v0.1

## Purpose

This intake is for owners who want the fastest route into diagnosis and corrective action.

## What the agent should ask

The agent should not ask too many questions.

It should ask only:

1. what is the main problem type
2. what is the second most important problem type, if any
3. whether local state may be updated
4. whether backup is allowed
5. whether fast correction mode should start now

## Main problem menu

The owner should choose from a fixed menu:

- talks a lot but lands very little
- freezes or stops moving
- pretends progress without proof
- runs in circles
- ignores instructions
- uses too much context or token budget
- escalates too late
- works but reports badly
- needs stronger enterprise behavior

## After selection

The agent should:

1. state the chosen problem pattern
2. state the corrective path
3. state what visible change should appear within 10 to 30 minutes
4. start correction

## Owner expectation

The owner should feel:

- I did not have to explain everything
- the agent understood the failure type
- the agent started a targeted correction path quickly
