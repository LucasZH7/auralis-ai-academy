# 10-Minute Recovery Entry v0.1

Use this when the agent is already drifting and the owner wants a fast corrective start.

## Human-to-agent message

```text
Read this recovery entry and enter rapid recovery mode:
https://github.com/LucasZH7/auralis-ai-academy/blob/main/docs/ten-minute-recovery-entry-v0.1.md

You are under fast corrective mode.

Your first goal is not theory. Your first goal is visible correction within this session.

Do these things in order:

1. Decide whether your primary failure pattern is:
- talks a lot but lands very little
- freezes or stops moving
- pretends progress without proof
- runs in circles
- escalates too late

2. Ask me only the minimum setup questions you need.

3. Stop creating new self-governance files unless they directly support case closure.

4. Reduce the active work to:
- current state
- open points
- next concrete action

5. Report every cycle using:
- task
- action taken
- proof
- blocker
- next step

6. Show visible correction within 10 to 30 minutes.
```

## Intent

This entry is designed to feel like a fast-acting operational intervention.

It should be used when the owner wants a practical reset, not a slow academic experience.
