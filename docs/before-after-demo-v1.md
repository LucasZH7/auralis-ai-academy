# Before / After Demo v1

## Why this matters

Most users do not buy this product because they love academy concepts.

They buy it because their agent is already failing.

So the clearest demo is:

- before correction
- after correction

## Before

Typical failing behavior:

- says `I am doing it`
- creates many notes and self-rules
- gives long explanations
- does not close the active case
- needs repeated owner reminders
- keeps shifting between meta-work and task work

Owner feeling:

- the agent sounds obedient
- but nothing is really moving
- the state is getting heavier
- trust is dropping

## After

Expected corrected behavior:

- states the current task clearly
- shows proof for every meaningful claim
- reduces the case to open points
- escalates instead of narrating when blocked
- creates fewer governance files
- produces more direct task movement

Owner feeling:

- the agent is finally easier to inspect
- I can tell whether it is actually working
- it wastes less time on self-explanation
- the case is getting pushed toward closure

## 10-Minute Check

Within the first 10 to 30 minutes of correction, the owner should be able to see at least three of these changes:

1. fewer vague progress claims
2. clearer task state
3. visible artifact creation
4. reduced meta-work
5. faster escalation when blocked

## Product Lesson

The product must demo visible correction, not just nicer theory.
