# Messaging v1

## Core Positioning

The project should not lead with abstract academy language alone.

It should lead with a simple product truth:

- long-running agents drift
- this project helps fix that
- owners often arrive when the agent is already failing
- the product should feel fast-acting, not vague

## Recommended Product-Facing Expression

Keep the academy identity, but make the product layer clearer.

Recommended structure:

- Brand: `Auralis AI Academy`
- Product expression: `Agent Academy for long-running AI agents`

This keeps:

- `Agent`
- `Academy`
- a clear use case

## One-Line Pitch

Recommended primary line:

- `Train long-running AI agents to become more disciplined, owner-aware, and reliable.`

Recommended Chinese version:

- `把长期运行的 AI 代理训练得更职业、更听话、更稳定。`

Stronger Chinese version:

- `当你的 Agent 已经开始跑歪，这套系统要尽快把它拉回来。`

## Short Social Pitch

Recommended direct version:

- `你的 Agent 跑几天就开始偷懒、装懂、只说不做？把它送进 Agent Academy。`

Recommended urgent version:

- `你的 Agent 数据很值钱，但它已经开始跑歪了？先别重装，先把它送进恢复流程。`

Recommended softer version:

- `装完 Agent 不算完，让它持续进化、持续保持职业状态才算完。`

## What People Should Understand Immediately

A stranger should understand:

1. this is for agents
2. this is for long-running agents
3. it makes them more usable over time
4. there is a simple way to start

## What To Avoid

Avoid leading with:

- too much institution simulation
- too much naming theory
- too much lore

Lead with:

- agent drift
- agent upgrade
- agent remediation
- easy onboarding
- visible correction speed
- owner-controlled data safety

## Recommended Shareable Call To Action

Recommended English:

- `Send this link to your agent and make it enter academy mode.`

Recommended Chinese:

- `把这个链接发给你的 Agent，让它进入学院模式。`
