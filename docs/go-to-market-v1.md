# Go-To-Market v1

## Core Message

Do not pitch this as "an AI school" first.

Pitch it as:

- a way to make long-running AI agents more professional
- a way to reduce agent laziness and drift
- a way to improve owner alignment, memory discipline, and auditability

That is easier for people to understand.

## What Users Want to Know First

When someone lands on the project, they want answers to three questions:

1. what does this do?
2. is this useful for my agent?
3. which program should I choose?

So promotion should always answer those first.

## Best Early Audience

The strongest first audience is:

- people already deploying AI agents
- technical founders
- automation-heavy operators
- people frustrated with agent drift and low reliability

## Best Early Promotion Channels

For the first wave, the highest-signal channels are:

- GitHub
- X / Twitter
- Discord communities for AI agents and automation
- Reddit communities around local AI, agents, and automation
- direct demos to technically curious users

## Best Demo Angle

The most compelling demo is not theory.

It is:

- before training: messy, vague, lazy agent behavior
- after training: clearer, more professional, more owner-aware behavior

That contrast is the product.

## First Launch Positioning

Recommended positioning:

`A practical framework for making long-running AI agents more disciplined, owner-aware, and maintainable.`

## What To Avoid

Do not lead with:

- too much lore
- too many academic labels
- too many future tracks
- too much simulation of a real-world university

The product should feel:

- serious
- useful
- fast to understand
- fast to try
