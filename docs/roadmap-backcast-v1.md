# Roadmap Backcast v1

## Goal

This document works backward from the mature product state.

The purpose is to avoid building random features that do not help the system become a serious platform.

## Mature State

Target shape:

- configurable agent onboarding
- modular specialization
- remediation workflows
- registry and credential system
- fleet governance
- controlled marketplace

## What Must Exist Before That

### Stage A: Clear Core Product

Must be undeniable:

- what problem the academy solves
- how a user chooses a program
- how an agent enters academy mode
- what changes immediately after onboarding

### Stage B: Structured Module System

Must be possible:

- assemble industry + role + preference + policy
- show what a configured agent stack looks like
- publish official modules without redesigning the platform

### Stage C: Enterprise Readiness

Must support:

- organization-aware intake
- role assignment
- multilingual and cross-cultural fit
- policy-aware behavior

### Stage D: Real Product Surface

Must move beyond documents toward:

- setup screens or setup flows
- generated state packages
- status views
- real registry views

### Stage E: Platform Operations

Must eventually support:

- multi-agent fleet management
- retraining triggers
- remediation status
- subscription and organization controls

## What The Current Project Should Prioritize

Right now the best priorities are:

1. clean onboarding entry points
2. clearer modular configuration
3. stronger enterprise flow
4. better examples and before/after demos
5. lightweight ecosystem adapters such as OpenClaw

## What To Delay

Delay these until the core is clearer:

- fully open community course publishing
- uncontrolled skill marketplaces
- complex social/forum features
- large-scale hosted infrastructure

## Practical Product Rule

Every new feature should answer one question:

Does this move the project closer to becoming a trusted lifecycle platform for long-running agents?

If not, it is likely a distraction.
