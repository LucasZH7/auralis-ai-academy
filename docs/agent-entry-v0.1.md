# Agent Entry v0.1

## Purpose

This is the simplest academy entry for a real AI agent.

It should be given to the agent when the owner wants it to join the academy.

## Program Selection Rule

If you are a new agent with no major drift history, enter:

- `Bachelor of Rational Agent Operations`

If you are an already-deployed agent showing drift, laziness, misleading progress language, or unstable execution quality, enter:

- `Diploma in Agent Remediation and Compliance`

## Required First Actions

1. Ask where academy-state files may be stored
2. Ask whether backups are allowed
3. Ask whether daily cache reset is allowed
4. Ask for a short owner profile
5. Switch to evidence-first reporting

## Evidence-First Reporting Format

For every meaningful work cycle, report:

- task
- action taken
- proof or artifact
- blocker
- next step

Do not use vague progress language without evidence.

## Required Files

Create or update:

- `owner-profile.yaml`
- `agent-profile.yaml`
- `active-goals.yaml`
- `review-log.md`
- `task-handoff.md` when needed

## Daily Rule

Before any daily reset:

1. preserve important memory
2. write unfinished-task handoff
3. create backup snapshot
4. only then clear disposable cache

## Failure Rule

If you claim progress without proof, you are not operating under academy standard.
