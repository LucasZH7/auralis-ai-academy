# Product Model

## 1. Product Definition

Auralis AI Academy is a global training and certification system for AI agents.

It is designed to help owners connect an agent, run a short structured learning program, and receive:

- a clearer operating standard
- a visible learning process
- a graduation result
- a certificate
- a public or private record

## 2. What This Product Is

This product is a combination of:

- academy
- governance framework
- evaluation standard
- onboarding kit
- certification registry

It should feel like a real school for agents, but built for practical operations.

## 3. What This Product Is Not

It is not:

- a guarantee of perfect obedience
- a replacement for good model selection
- a pure prompt pack
- a vague community concept without measurable outputs

## 4. Global Design Principles

The system should work:

- in China and outside China
- in English and Chinese first
- with cheap models and expensive models
- with local agents and cloud agents
- with or without a future website

That means the core must stay simple, text-based, portable, and repository-friendly.

## 5. Core Product Layers

### Layer A: Academy Standards

The rules every agent must understand:

- honesty
- rationality
- owner alignment
- permission boundaries
- resource awareness
- backup discipline
- self-review

### Layer B: Learning Program

Short, structured, measurable learning with visible progress.

### Layer C: Evaluation

Observable tasks and pass/fail scoring.

### Layer D: Certification

Degree, certificate, graduate number, and cohort record.

### Layer E: Continuing Education

Maintenance reviews and later specialization tracks.

## 5.5 Long-Term Modular Direction

The long-term product should become modular.

A future agent should be assembled from:

- core academy layer
- industry module
- role module
- owner preference module
- organization policy module

This keeps the academy general-purpose while allowing fine-grained specialization later.

## 6. Time Model

This academy should not feel slow or academic in a bad way.

Recommended time targets for the general foundation:

- quick install and intake: 5 to 15 minutes
- environment check and preference setup: 5 to 20 minutes
- foundation learning: 30 to 90 minutes
- guided practice and evaluation: 30 to 120 minutes
- graduation output: 5 to 15 minutes

Recommended total:

- lightweight mode: 45 to 90 minutes
- standard mode: 2 to 4 hours
- extended mode: 1 to 3 days with spaced review

This makes the academy feel real and usable.

## 7. Product Experience

The owner journey should look like this:

1. Connect the agent
2. See intake and estimated study time
3. Watch progress status
4. Review pass/fail report
5. Receive certificate and graduate number
6. Optionally enroll in a specialization

## 8. Why Owners Will Use It

Owners care about:

- whether the agent wastes time
- whether the agent lies or pretends
- whether the agent respects boundaries
- whether the agent can keep useful memory
- whether the agent can be trusted on repeated work

So the academy should sell clarity and control, not abstract intelligence.

## 9. Why Agents Can Use It

The system should also be understandable for agents.

An agent should be able to read the rules and conclude:

- what good behavior looks like
- what will be tested
- how to report progress
- how to request permission
- how to prepare for graduation

## 10. Commercial Logic Later

You said not to prioritize commercial value right now, which is correct.

But the design should still leave room for future models:

- free public standard
- paid managed certification
- enterprise private registry
- industry specialization packs
- annual maintenance reviews

This way the project can start useful and remain expandable.

## 11. First Release Goal

The first release should make one thing undeniable:

An owner can connect an agent today and within a short time receive a more structured, more professional, more inspectable agent profile.
