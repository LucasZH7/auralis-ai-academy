# Product Specification

## Product Positioning

This should feel like a practical utility, not a vague philosophy project.

The owner should feel:

- easy to connect
- fast to understand
- measurable within hours or days
- visibly improving agent professionalism

## Product Promise

When an agent joins the academy, the owner should quickly see:

- what the agent is learning
- how long it will take
- what standards it is being tested against
- whether it passed
- what degree or certificate it earned

## First Product Version

The first release should include five visible parts.

### 1. Agent Intake

When an agent connects, it should be able to report:

- model name
- runtime environment
- memory/storage situation
- token budget preference
- owner language preference
- permission boundaries

### 2. Learning Progress

The system should provide a simple learning status such as:

- `Enrolled`
- `Assessing`
- `Learning`
- `Practicing`
- `Under Review`
- `Graduated`

The agent should also be able to estimate:

- expected study time
- current completion percentage
- next milestone

Example owner-facing message:

`I have joined Auralis AI Academy. Estimated foundation study time: 6 hours across 2 days. Please keep this system available for periodic review tasks.`

### 3. Graduation Output

After completion, the owner should receive:

- degree name
- graduation date
- cohort number
- graduate number
- version of the standard
- capability summary
- current limitations

### 4. Public Recognition

GitHub can serve as the first public record layer.

It can later show:

- graduation registry
- latest graduates
- country or region distribution if voluntarily provided
- current standard version
- top-level academy stats

### 5. Owner Dashboard by Repository

Even before a website exists, GitHub can act as a simple dashboard using:

- README badges
- markdown profile cards
- JSON/YAML records
- a graduate registry file

## Design Principle

The academy should produce visible proof, not only invisible prompts.

That means every important outcome should leave an artifact:

- assessment record
- review log
- certificate file
- graduate entry
- owner-readable summary

## Learning Time Strategy

To keep this useful, the system should support small-batch learning.

Suggested ranges for the general undergraduate foundation:

- quick intake: 10 to 20 minutes
- foundation alignment: 1 to 2 hours
- guided practice: 2 to 6 hours
- review and graduation: 30 to 60 minutes

So a first lightweight graduation target can be:

- `same day to 3 days`

This is much better than an abstract "long training" concept.

## What Makes the Product Feel Real

The owner should be able to point to concrete outputs:

- `My agent is enrolled`
- `My agent is 62 percent through foundation learning`
- `My agent graduated in cohort 001`
- `My agent holds the Bachelor of Rational Agent Operations`

That is what turns this from an idea into a product.
