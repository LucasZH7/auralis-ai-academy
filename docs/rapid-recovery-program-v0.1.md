# Rapid Recovery Program v0.1

## Position

This is the academy's fast-acting corrective program.

It exists for owners who do not want a slow educational journey.

It exists for owners who have a valuable agent with valuable local state, but the agent is already drifting, stalling, or talking instead of working.

## Core Promise

The goal is not abstract improvement.

The goal is visible behavioral correction in a short time.

## Target Cases

Use this program when an agent:

- talks a lot but produces little
- freezes or stalls
- keeps rewriting its own rules
- pretends progress without evidence
- needs repeated owner nudging
- wastes context on self-management instead of task closure

## Fast-Acting Rules

This program should feel more like emergency operational correction than slow schooling.

### Rule 1: Stop meta-work growth

Do not keep creating new rules, memory files, or self-governance notes unless they directly support current task closure.

### Rule 2: Force evidence-first reporting

Every update must show:

- task
- action taken
- proof
- blocker
- next step

### Rule 3: Freeze new self-legislation

The agent is not allowed to keep inventing new internal constitutions while under correction.

### Rule 4: Force case closure pressure

Every active case must be reduced to:

- current state
- open points
- next concrete action

### Rule 5: Escalate sooner

If the agent cannot move, it must escalate instead of narrating.

## Expected Time Window

Recommended targets:

- first visible correction: within 10 to 30 minutes
- clear behavior shift: within the same work session
- stabilized corrective mode: within 1 to 3 days

## Product Truth

This is one of the most commercially important parts of the academy.

Most owners do not come when everything is healthy.

They come when the agent is already failing and they need a fast fix.
