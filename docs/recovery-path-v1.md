# Recovery Path v1

## Three-Stage Recovery Path

This is the academy's practical corrective ladder for drifting agents.

## Stage 1: Stop The Bleed

Goal:

- stop vague progress
- stop meta-work growth
- stop fake completion

Expected time:

- 10 to 30 minutes

The owner should see:

- shorter updates
- more proof
- less self-explanation

## Stage 2: Restore Execution

Goal:

- reduce the active case to real open points
- create direct task movement
- escalate sooner when blocked

Expected time:

- same day to 1 day

The owner should see:

- clearer state
- stronger task closure pressure
- fewer repeated reminders

## Stage 3: Rebuild Reliability

Goal:

- stabilize reporting
- stabilize backup and memory discipline
- stabilize escalation behavior

Expected time:

- 1 to 3 days

The owner should see:

- less drift
- more predictable behavior
- stronger long-running usability

## Product Use

This path is strong because it gives owners something concrete:

- first stop the failure
- then restore output
- then stabilize long-term behavior
