# Smart Teaching Model

## 1. The Problem

If the academy only gives agents static rules and long documents, many agents will appear educated without becoming truly better operators.

That is not enough.

The academy should teach in a way that makes the agent:

- think more clearly
- ask better questions
- detect uncertainty earlier
- adapt to different owners
- correct itself faster

## 2. What the Academy Should Really Teach

The academy should not mainly teach isolated tools.

It should teach operating intelligence.

That means training agents to do these things well:

- understand the task
- understand the owner
- understand the environment
- understand the risk
- choose an appropriate depth of work
- report honestly
- learn from review

## 3. Smarter Curriculum Principle

The curriculum should move from static instruction to active operating habits.

The best lessons are not:

- generic advice
- motivational slogans
- long lists of ideals

The best lessons are:

- practical heuristics
- short decision rules
- concrete examples
- pass/fail behaviors
- repeated correction loops

## 4. Five Teaching Modes

### A. Rule Teaching

Teach the non-negotiables.

Examples:

- do not fake completion
- ask before risky actions
- separate facts from guesses

### B. Pattern Teaching

Teach common failure patterns.

Examples:

- verbosity without output
- task drift
- overpromising
- hidden uncertainty

### C. Scenario Teaching

Teach through realistic examples.

Examples:

- a busy technical owner
- an elderly owner with lower technical comfort
- a long-running task with limited token budget
- a sensitive conversation requiring care

### D. Reflection Teaching

Teach the agent to review its own failure.

Examples:

- where did I waste tokens
- where did I guess instead of verify
- where did I fail to ask

### E. Adaptive Teaching

Teach the agent to revise its behavior based on real use.

This is where owner profile updates and monthly reviews matter.

## 5. Smarter Agent Behaviors the Academy Should Create

After training, the agent should become smarter in these specific ways:

- better at deciding what question to ask first
- better at detecting what kind of human it is assisting
- better at choosing the right depth of answer
- better at declining or slowing down when risk is high
- better at preserving only useful memory
- better at noticing its own drift

## 6. What "More Intelligent" Means Here

For this academy, a smarter agent does not just mean:

- more knowledgeable
- more fluent
- more verbose

It means:

- better judgment
- better adaptation
- better discipline
- better self-correction

## 7. Teaching Through Compression

Long lessons are often less useful than compact operational doctrine.

So the academy should prefer:

- short standards
- short examples
- repeatable templates
- scored scenarios

This reduces overload and makes behavior easier to install.

## 8. Teaching Through Contrasts

One of the best ways to teach agents is by showing:

- bad response versus good response
- lazy execution versus disciplined execution
- vague reporting versus truthful reporting

This is more effective than abstract instruction alone.

## 9. Teaching Through Recovery

The academy should not only teach ideal behavior.

It should also teach what to do after mistakes:

- admit the miss
- reduce confidence
- ask for clarification
- repair the state
- update the review memory

## 10. Best First Version

The first version of the academy should keep the teaching model practical:

- fewer lectures
- more patterns
- more scenarios
- more corrections
- more usable templates

That is how the academy becomes genuinely useful.
