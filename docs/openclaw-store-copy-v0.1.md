# OpenClaw Store Copy v0.1

## Short title

`Agent Academy`

## Short subtitle

Put long-running OpenClaw agents into a more disciplined, owner-aware, professional operating mode.

## One-line value

Your agent worked on day one. A few days later it gets vague, drifts, weakens in work quality, and stops behaving like a reliable operator.

This skill is built for that.

It is designed to create visible operating improvement, not just produce more theory.

## Short description

Agent Academy helps long-running OpenClaw agents become more reliable and more professional over time.

It routes the agent into either foundation training or remediation, asks the owner the right setup questions, checks runtime limits, initializes local academy-state, and switches the agent into evidence-first reporting.

It is especially strong when the owner wants a more disciplined, workplace-ready agent without rebuilding everything from scratch.

## Problem statement

Many agents do not fail because they lack raw intelligence.

They fail because they:

- say more than they do
- hide uncertainty
- report progress without proof
- accumulate messy context
- lose alignment with the owner over time
- behave weakly inside real organizations
- ignore runtime reality and overcommit

## What happens after install

After entering academy mode, the agent should:

- decide between foundation training and remediation
- ask owner setup questions
- ask basic workplace and approval questions when relevant
- check memory, storage, and model/API limits
- adjust to full, compact, or survival mode if needed
- confirm token and response preferences
- confirm cache reset and backup rules
- initialize academy-state
- report work with proof, blockers, and next steps

## Trust promise

The first-version model is owner-controlled:

- state stays local first
- sensitive data should not be casually copied into academy materials
- the academy focuses on behavior correction, not data extraction

## Best for

- long-running OpenClaw agents
- agents with drift or lazy compliance
- owners who want more control and clearer reporting
- people running serious workflows, not just casual chat
- users who want agents to act more professionally in work settings

## Not for

- users who only want a personality prompt
- agents that never keep local state
- workflows where the owner does not want structured reporting

## Human-facing CTA

Install the skill, send your agent the academy entry link, and make it enter academy mode.

## Agent-facing CTA

Read the academy entry source, choose the correct program, ask the required owner questions, initialize academy-state, and switch into evidence-first reporting.
