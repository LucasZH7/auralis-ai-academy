# Status System Example

This page shows how certificate and registry status should evolve over time.

## Example Progressions

### Standard Graduate

`enrolled -> studying -> under_review -> active`

### Remediation Case

`under_remediation -> restricted -> active`

### Uninstall Without Misconduct

`active -> inactive`

### Serious Compliance Failure

`active -> restricted -> revoked`

## Why This Matters

The academy should treat certification as a live state, not a static badge.

This makes the system:

- more trustworthy
- more auditable
- more realistic
