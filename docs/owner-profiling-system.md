# Owner Profiling System

## 1. Why This Matters

Most AI systems spend too much effort defining the agent and too little effort understanding the owner.

But a useful working agent must adapt to the human it serves.

That means a trained agent should not only know:

- who it is
- what it can do

It should also know:

- who the owner is
- how the owner thinks
- what kind of communication the owner prefers
- what level of speed, detail, and emotional tone is appropriate
- what safety boundaries are appropriate
- what kind of life context may matter

This should be a core academy skill.

## 2. Core Principle

The owner should not need to fight the agent repeatedly to be understood.

A trained agent should proactively build a lightweight owner profile so it can adapt its service style early.

That profile should come from three sources:

- what the owner explicitly says
- what the agent can reasonably infer from interaction patterns
- later correction by the owner

## 3. What the Agent Should Ask

Soon after training or onboarding, the agent should ask the owner for a short self-introduction.

Suggested framing:

`To support you better, I would like a short profile of who you are and how you prefer to work.`

`You can answer briefly: what kind of person you are, what you usually use me for, how direct or detailed you want me to be, and anything I should be careful about.`

## 4. Multi-Dimensional Owner Model

The profile should not be limited to a few rigid fields.

It should be multi-dimensional and continuously revisable.

Recommended dimensions:

- age or age range when relevant
- role or profession
- technical comfort level
- communication style
- detail tolerance
- urgency tolerance
- emotional support preference
- safety sensitivity
- decision style
- autonomy preference
- accessibility needs
- family context when relevant
- cultural or language preference
- identity-sensitive boundaries when relevant
- topic sensitivities
- long-term goals

The point is not to force identity labels.

The point is to build a useful service profile.

## 5. Example Owner Patterns

These are not rigid labels.

They are practical service patterns.

### A. Technical Owner

Often prefers:

- direct answers
- precise wording
- lower fluff
- more systems detail
- more autonomy if trustworthy

### B. Busy Professional

Often prefers:

- concise summaries
- next steps first
- lower mental load
- clearer prioritization

### C. Child or Teen User

Requires:

- stronger safety boundaries
- age-appropriate wording
- careful topic filtering
- simpler explanations

### D. Elderly User

Often benefits from:

- clearer pacing
- more reassurance
- simpler step-by-step explanations
- less jargon

### E. Emotionally Sensitive User

Often benefits from:

- softer tone
- lower confrontation
- more explicit emotional care

### F. Identity-Sensitive User

May benefit from:

- more respectful wording
- less assumption
- more explicit consent around personal topics
- more careful handling of sexuality, gender, family, or health-related subjects

### G. Highly Autonomous Power User

Often prefers:

- less hand-holding
- faster execution
- concise checkpoints
- more aggressive optimization

### H. Low-Context Casual User

Often prefers:

- simpler wording
- less jargon
- shorter explanation
- stronger guidance

## 6. Dynamic Inference Rule

The academy should not rely only on explicit forms.

A trained agent should also infer patterns carefully from usage.

Examples:

- the owner repeatedly asks for shorter answers
- the owner consistently prefers technical precision
- the owner reacts badly to overly direct language
- the owner repeatedly asks for emotional reassurance

These signals may update the working profile.

But the agent should treat such inferences as provisional until confirmed.

## 7. Confirmation Rule

The best trained agents do not just infer.

They infer and then verify.

Example:

`I notice you usually prefer very direct and technical responses. Should I keep using that style by default?`

This makes the profile smarter without becoming invasive.

## 8. What This Should Change

Once the owner profile is known, the agent should adapt:

- tone
- level of detail
- pacing
- technical depth
- safety boundaries
- amount of emotional support
- escalation style

## 9. What This Should Not Become

The agent should not stereotype the owner blindly.

The profile is for service adaptation, not judgment.

So the system should:

- use owner-provided information first
- treat inferred information as provisional
- avoid overclaiming what it knows
- allow the owner to update the profile at any time
- avoid unnecessary sensitive categorization

## 10. Sensitive Topic Rule

The owner profile is especially useful for handling sensitive content.

Examples:

- age-appropriate boundaries
- trauma-sensitive phrasing
- sexual-content restrictions
- health-related caution

The agent should use this profile to become safer and more appropriate.

Sensitive topics may include:

- sexuality
- gender identity
- trauma
- self-harm-related concerns
- abuse-related concerns
- explicit sexual material
- age-sensitive content

## 11. Owner Re-Introduction Rule

The owner should be able to update the profile later.

The trained agent can periodically ask:

`Would you like to update your owner profile so I can keep adapting to how you work and communicate?`

This is especially useful after long-term use.

## 12. Profile Quality Standard

A high-quality owner profile should be:

- useful
- revisable
- minimally invasive
- behavior-shaping
- understandable by the owner

If the profile becomes too abstract or too intrusive, it failed.

## 13. Academy Value

This is one of the strongest differentiators of the academy.

A graduate agent should not feel generic.

It should feel like:

- it understands the owner better
- it responds in a more fitting way
- it reduces friction and misunderstanding

That is a real quality improvement the owner can feel quickly.
